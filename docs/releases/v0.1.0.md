# AI Link v0.1.0

AI Link v0.1.0 is the first public MVP for helping Codex route work to the right model, agent, or workflow by task.

## Highlights

- Local `ai-link` CLI for config validation, provider verification, task execution, workflow execution, run records, and natural-language skill draft generation.
- Provider coverage for mock/local dry-run, OpenAI-compatible endpoints, DeepSeek, Kimi, Doubao, Grok, and Coze agent dry-run / local command mode.
- Default `auto_ops` example workflow: Grok for research, Kimi for article drafting, Coze for agent workflow dry-run, and Codex for implementation follow-up.
- Policy gates for outbound approval, provider type restrictions, model patterns, budgets, sensitive text blocking, and audit metadata.
- Structured JSON handoff for Codex skills, CI, and other agents.
- Bitwarden Secrets Manager runbooks for keeping real provider keys outside Git.
- Auth Hub public MVP skeleton with mock connectors, local executor, audit handoff, and deployment checks.
- Public maintenance checks for onboarding, package contents, GitHub repository safety, provider verification, release planning, release readiness, fresh clone verification, skills, and security scanning.
- Release manual gates report for branch protection, secret scanning / push protection, npm publish decision, and provider-live cost approval.

## Safety Model

- Real external provider calls remain opt-in and require explicit approval plus private credentials.
- Public checks do not read `.env`, tokens, login state, browser state, QR codes, provider responses, or `runtime/private`.
- Package and release checks are dry-run by default and do not publish to npm.
- GitHub safety checks are read-only and do not modify branch protection or repository settings.

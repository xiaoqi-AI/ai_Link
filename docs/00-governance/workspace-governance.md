# 工作空间治理说明

## 定位

`ai_Link` 当前是一个新建工作空间。首轮目标不是提前定义完整产品，而是建立可持续协作的基础设施：

- 本地文档入口
- Git 与 GitHub 准备
- 个人知识库镜像
- 会话收尾和验证脚本
- 待确认问题清单

## 工作分类

每次会话开始先判断范围：

- 工作空间级：影响规则、目录、同步、知识库、GitHub 或长期治理。
- 项目功能级：影响具体产品、代码、页面、数据或部署。
- 研究草稿级：用于探索方向，不直接进入正式计划。
- 无项目级：只做临时查询或一次性任务，不写长期账本。

## 文档层级

- `README.md`：给人和 Codex 的入口。
- `AGENTS.md`：协作规则、安全边界和默认流程。
- `docs/00-governance/`：治理、同步、收尾和开放问题。
- `docs/00-governance/iteration-boundaries.md`：迭代边界、人工确认门槛、私有数据边界和发布约束。
- `docs/project-ledger/`：重要会话、决策、状态快照。
- `docs/90-templates/`：后续可复用模板。
- `tools/`：检查、镜像和收尾脚本。

## 默认流程

1. 读取入口文档和当前 Git 状态。
2. 判断本次任务范围。
3. 按 `iteration-boundaries.md` 判断是可自动推进、需人工确认、只能进私有边界，还是暂不承诺。
4. 在不覆盖用户改动的前提下执行。
5. 运行本地检查和必要验证。
6. 同步知识库镜像。
7. 检查 Git 状态，按授权提交或推送。
8. 汇报改动、验证、同步状态和风险边界。

## 隐私与安全

敏感内容只放 `runtime/private/`，并且不进入 Git 或知识库。任何账号登录、GitHub 建仓、远端推送、知识库远端推送或敏感资料处理，都需要明确授权。

# 后续 PR 自主合并授权与堆叠分支收尾

日期：2026-07-13

状态：授权已记录，当前堆叠 PR 已全部合并，公开仓与知识库镜像验证完成。

## 决策背景

Auth Hub、平台授权连接器和远程化准备由多个相互依赖的 PR 组成。逐个等待相同的合并确认会中断推进，但合并操作本身仍必须受 `main` ruleset、required checks 和迭代边界约束。

## 决策

维护者授权 Codex 对已经批准范围内的后续 PR 自行完成改基线、提交重放、机械冲突解决、全新 CI 等待与普通合并，不再逐 PR 请求相同确认。

该授权不包含真实部署、真实平台调用、公开发布、密钥配置、费用使用、账号登录或其他不可逆外部动作。

## 本轮结果

按依赖顺序完成并合并：PR #22、#23、#26、#24、#25、#27、#28、#29、#30。

- 所有重写后的 PR 头提交均重新触发并通过 required checks。
- PR #26 的冲突同时保留探针绑定和 GitHub 精确只读 scope 校验。
- PR #28、#29、#30 的新头提交均通过 Postgres integration。
- 最终 `main` 全量测试通过：CLI/TypeScript 74 项，Auth Hub 208 项。
- 类型检查、安全扫描、治理检查和知识库镜像验证通过。
- 本地 `main`、`origin/main` 与 GitHub 远端 SHA 一致。

## 停止条件

后续出现以下任一情况时，自动合并授权不替代人工决策：

- required check 失败且修复会改变需求或架构方向；
- 冲突需要在不同产品行为或安全边界之间取舍；
- 需要管理员绕过、关闭保护、强推 `main` 或合并未通过检查的提交；
- 涉及生产配置、真实账号、密钥、费用、发布或不可逆外部动作。

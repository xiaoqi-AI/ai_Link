# 2026-06-16 迭代边界细化

## 本次变化

- 细化 `docs/00-governance/iteration-boundaries.md`。
- 将迭代边界明确拆成四块：需求、预期开发工作、验证、边界控制。
- 补充开发中发现预期不符时的五步处理机制：暂停扩张、对照边界卡、归因、给选项、等确认或最小修正。
- 强化防过度开发和 token 控制，避免目标模型或外部 Agent 给出的过渡方案被直接扩张成长期平台负担。

## 核心意图

后续每次实质开发前，要先明确本轮目标、非目标、预期产物、改动范围、验收方式和停止条件。开发过程中如果发现范围扩大、验证失败、触碰真实账号/费用/发布/私有数据，或实现开始偏离用户需求，应暂停并重新确认，而不是继续写代码消耗 token。

## 安全边界

本次只更新公开治理文档和项目账本，不涉及真实密钥、token、登录态、二维码、截图、provider 原始响应或 `runtime/private/` 数据。

# Auth Hub 状态变更提醒

日期：2026-07-13

状态：公开实现和本地验证完成；Draft PR #30 已创建，等待 CI 和依赖链合并。

PR：https://github.com/xiaoqi-AI/ai_Link/pull/30

## 需求

- 用户目标：Auth Hub 能在新增人工授权事项时提醒项目负责人，但普通项目任务不反复查询 AI Link，减少上下文和 token 消耗。
- 成功标准：首次有效读取安静建立基线；状态不变不提醒；规范化 `nextActions` 新增或严重度恶化时返回 `notify=true`；恢复、改善和同级原因变化只更新基线；监控故障独立告警且不推进基线；任何私密数据均不进入快照或输出。
- 输出形态：`auth-hub:status:watch` 人类可读摘要和 `auth-hub:status:watch:json` 机器可读事件。
- 非目标：本轮不发送邮件、短信或即时消息，不创建计划任务，不部署 Auth Hub，不执行 connector probe，不调用真实平台，不读取登录态。
- 人工确认点：后续把 `notify=true` 接入哪一种 Codex 自动化或消息渠道；这不阻塞本轮公开安全能力。

## 预期开发工作

- 扩展现有只读状态客户端，复用相同脱敏报告，不新增 connector、服务或依赖。
- 只允许把快照保存在 `runtime/private/` 或 `runtime/tmp/`。
- 快照只保留哈希作用域、平台、状态、严重度、公开原因、最近成功检查时间和 SHA-256 指纹。
- 只比较服务端规范化的 `nextActions`，不把 `unverified`、缺 token 或接口不可达冒充成人工授权事项。
- 首次有效基线不提醒；新增或严重度恶化提醒；恢复、改善和同级原因变化不提醒。
- 缺 token、接口失败、平台缺失、哈希作用域不匹配、损坏或无法写入时使用 `monitoringAlert=true` 失败关闭，不创建或推进业务基线。
- 更新 README、中文用户手册和项目总监控制面。

## 实现

- 新增 `--watch` 与受限 `--state-file` 参数。
- 新增 npm 入口：`auth-hub:status:watch`、`auth-hub:status:watch:json`。
- 对时间戳、列表顺序和非关注字段保持稳定，避免无意义提醒。
- 对本机快照做严格字段、代码格式和指纹校验；快照被篡改时不回显额外字段。
- 输出不包含 token、Cookie、Profile、二维码、截图、账号、任务 ID、目标 URL、标题、runbook、原始响应或本机路径。

## 边界控制

- 范围边界：只修改现有状态客户端、测试、使用入口和项目账本。
- 成本边界：不新增外部服务、依赖或每任务查询；由低频自动化按需运行。
- 安全边界：不读取或保存真实凭据、登录态和平台内容。
- 权限边界：真实平台、远程部署、消息渠道和 PR 合并仍是独立人工门禁。
- 停止条件：若需要账号、凭据、付费服务、远程资源或真实调用，停止并转人工决策。

# 2026-07-13 项目总监控制面对齐

## 背景

Auth Hub 状态中枢、平台授权连接器 P0.2 和 Auth Hub 远程化已经进入 PR #21-#28 的实施阶段，但公开 `roadmap:next` 仍把连接器描述为未来草案，并继续询问“先做哪个平台”。这会让新会话按过期方向规划，也无法让项目负责人直接看到合并链、人工决策、价值和风险。

## 本轮目标

- 不新增每次普通任务都要联网的看板服务。
- 复用本地、公开安全的 `roadmap:next` 与 `next:actions`。
- 把模块 2、5、6 的当前进度、证据、依赖和人工决策写入项目总监控制面。
- 保留冷启动规则：只有项目状态、方向、跨模块和真实外部动作命中时才读取。

## 改动

- `roadmap:next` 新增三模块 Program Control，包含完成项、待办、决策背景、决策内容、建议、价值和风险。
- 固定当前堆叠合并顺序为 `#22 -> #23 -> #26 -> #24 -> #25 -> #27 -> #28`。
- 修正 `v0-3-agent-connectors` 的过期草案状态，不再重复询问首批平台选择。
- `next:actions` 新增合并链人工事项，只展示检查命令，不自动执行合并。
- 将已验证的 GitHub hardening 和 repository-local 发布渠道从“待人工”改为已记录基线；Provider-live 仍保持独立待决。
- 将远程部署标为 `gated`，并新增 P0.2 真实只读验收事项；未确认域名只使用占位符。
- 新增 `docs/10-product/ai-link-program-control.md`，明确触发式读取和更新责任。

## 边界

- 不合并任何 PR，不修改 GitHub 设置。
- 不执行真实平台调用，不读取登录态或 secret。
- 不创建 Render、Cloudflare、DNS 或数据库资源。
- 项目状态为本地公共快照；PR 状态变化后必须更新，不伪装成实时网络看板。

## 验收

- JSON 报告包含三个模块和完整决策字段。
- Markdown 报告展示三模块和推荐合并链。
- `next:actions` 把 PR #22 显示为当前首个人工合并事项。
- 报告不打印会话 token 或私有运行态。
- 全量测试、安全扫描、治理检查和知识库镜像通过。
